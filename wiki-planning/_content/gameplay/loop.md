# 核心循环

核心循环是玩家在 界限 中反复进行的基础游戏行为。一个好的核心循环应当**简单、有回报、可重复**。

## 主循环

```
        ┌──────────────────────────────────────┐
        │                                        │
        ▼                                        │
   选择武器 ──→ 战斗 ──→ 积累熟练度 ──→ 变强 ────┘
   (定义身份)   (核心)    (成长)        (回报)
        │                                        
        │  随时可换武器，切换流派                 
        └──────────────────────────────────────
```

### 循环四步

1. **选择武器（定义身份）**
   - 玩家手持某武器，系统识别其流派，套用对应属性与技能。
   - 这一步是"无成本"的——随时可换。

2. **战斗（核心行为）**
   - 用当前主武器击杀生物、挑战内容。
   - 战斗体验因流派而异：剑的连击、弓的远程、法杖的范围。

3. **积累熟练度（成长）**
   - 有效战斗为当前流派积累熟练度经验。
   - 经验只进当前主武器流派。

4. **变强（回报）**
   - 熟练度升级 → 属性提升 + 技能点 + 解锁新技能。
   - 玩家感受到"我练的武器更强了"。

## 循环的节奏

| 时间尺度 | 循环内容 | 玩家感受 |
|----------|----------|----------|
| 秒级 | 攻击、释放技能 | 操作爽快 |
| 分钟级 | 击杀一波怪、涨一点经验 | 稳步推进 |
| 小时级 | 升几级、解锁新技能 | 明显成长 |
| 天/周级 | 练满一个流派、转练新流派 | 长线目标 |

## 多流派的循环叠加

界限 的核心循环可以**并行多条**：

```
玩家 A 的循环状态：
  剑流派  ████████░░ Lv 42（主练）
  弓流派  ███░░░░░░░ Lv 15（副练）
  法杖    █░░░░░░░░░ Lv 3（尝鲜）
```

玩家可以随时切换正在推进的循环——今天练剑，明天练弓。每条循环独立积累，互不干扰。这让核心循环既有**深度**（单流派深耕）又有**广度**（多流派发展）。

## 循环的驱动力

是什么让玩家愿意一遍遍重复核心循环？

| 驱动力 | 来源 |
|--------|------|
| **即时反馈** | 每次攻击的打击感、技能特效 |
| **短期目标** | 下一级、下一个技能解锁 |
| **长期目标** | 练满流派、收集所有技能 |
| **新鲜感** | 随时切换流派换玩法 |

## 设计要点

::: tip 让每一步都有反馈
核心循环的每一步都应有清晰反馈：选武器时 HUD 显示流派切换，战斗时有打击感，涨经验时有进度条变化，升级时有提示动画。反馈密度决定循环的"爽快度"。
:::

::: warning 避免循环空转
如果战斗不给有效回报（如挂机刷怪），循环就会"空转"——玩家在重复但没成长感。防刷机制（见 [熟练度成长](/systems/mastery)）确保循环始终有意义。
:::

# 平衡原则

平衡原则是数值与玩法设计的最高准则。当具体数值需要取舍时，回到这些原则。

## 原则一：熟练度是成长主轴

> 玩家的强度主要来自熟练度成长，而非装备或运气。

- 装备、武器词条是"补充与特色"，不是"替代"。
- 一个高熟练度玩家用普通武器，不该弱于低熟练度玩家用神装。
- 这保证"用进废退"支柱：投入时间练习的玩家得到回报。

::: warning 反面案例
❌ 如果刷到传说武器就能碾压一切，玩家会去刷装备而非练熟练度，核心循环就崩了。
:::

## 原则二：流派强度相近，风格各异

> 没有"最强流派"，只有"最适合某场景的流派"。

- 每个流派在总体强度上应当接近。
- 差异体现在**擅长场景**：游侠擅长远程风筝，狂战擅长爆发，法师擅长群体。
- 玩家"因为喜欢而选"，而非"因为最强而选"。

### 平衡检验

| 检验问题 | 期望答案 |
|----------|----------|
| 是否有流派明显比其它强？ | 否 |
| 每个流派是否有擅长场景？ | 是 |
| 是否有流派毫无优势？ | 否 |

## 原则三：强力来自搭配，而非碾压

> 高强度应当来自技能、武器、属性的搭配组合，而非单一数值碾压。

- 鼓励玩家思考 combo（技能搭配、词条联动）。
- 避免"堆一个属性就无敌"的单一最优解。
- build 深度让游戏更耐玩。

## 原则四：自由不被惩罚

> 切换流派、尝试新玩法不应有数值惩罚。

- 练第二个流派不扣减第一个的进度。
- 切换武器不损失任何东西。
- 这保证"零锁定"支柱。

::: warning 与"专精锁"的冲突
有时为了平衡会想引入"专精锁"（限制只能强一个流派）。这违背零锁定原则，应当否决，改用其它平衡手段（如内容门槛用"任一流派达标"）。
:::

## 原则五：数值可控，防止膨胀

> 所有数值有合理上限，防止后期失控。

- 百分比属性（暴击、破甲、吸血等）设软/硬上限。
- 属性成长用线性或边际递减，避免指数爆炸。
- 终局武器强度提升有限且可控。

## 平衡的调节杠杆

当需要调平衡时，可动用的杠杆：

| 杠杆 | 调什么 | 影响 |
|------|--------|------|
| 经验曲线 | base / exponent | 成长速度 |
| 属性增量 | 每级属性成长 | 流派强度 |
| 技能数值 | 伤害/冷却/消耗 | 技能强度 |
| 武器词条 | 词条数值/概率 | 装备强度 |
| 属性上限 | 各属性软硬上限 | 防膨胀 |
| 内容门槛 | 副本/挑战等级要求 | 进程引导 |

## 平衡工作流

```
1. 发现问题（某流派过强/过弱、成长太快/太慢）
2. 定位杠杆（是属性？技能？曲线？）
3. 小步调整（避免大改导致连锁失衡）
4. 测试验证（实际游玩感受）
5. 迭代（重复直到平衡）
```

## 平衡决策检查清单

任何数值调整前，对照五原则：

- [ ] 是否维持了"熟练度为主轴"？
- [ ] 是否保持了流派间强度相近？
- [ ] 强力是否来自搭配而非碾压？
- [ ] 是否惩罚了玩家的自由选择？
- [ ] 数值是否在可控范围内？

## 平衡与核心支柱的关系

```
核心支柱（不可妥协）
  零锁定 > 用进废退 > 角色唯一
        ↓ 约束
平衡原则（数值层面的体现）
  熟练度主轴 / 流派相近 / 搭配制胜 / 自由无罚 / 数值可控
        ↓ 指导
具体数值（可迭代调整）
  曲线参数 / 属性增量 / 技能数值 ...
```

平衡服务于支柱。当平衡需求与核心支柱冲突时，**支柱优先**。

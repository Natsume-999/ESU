# 主武器机制

主武器机制决定玩家"当前是什么职业"。它是职业系统的入口。

## 识别方式：手持自动识别

> **设计决策**：采用手持自动识别，而非手动绑定。理由：最贴合"自由切换"理念——玩家无需任何命令或菜单操作，拿起武器即生效。

- 玩家**主手**持有的武器，即为当前**主武器**。
- 系统通过武器的**流派标签**判定其所属流派。
- 切换主手物品时，主武器流派实时切换，属性与技能同步更新。
- 手中无武器（空手或非武器）时，进入**徒手流派**。

```
玩家手持铁剑   → 识别为「剑」流派 → 套用剑系属性 + 剑系技能
玩家切换到弓   → 识别为「弓」流派 → 套用弓系属性 + 弓系技能
玩家收起武器   → 徒手流派
```

## 切换的即时性

主武器切换必须满足"零锁定"支柱：

| 要求 | 说明 |
|------|------|
| **即时生效** | 切换快捷栏的瞬间，流派就变了 |
| **无冷却** | 可以反复切换，没有切换 CD |
| **无代价** | 切换不消耗任何资源 |
| **可逆** | 随时切回，状态完全恢复 |

## 切换时发生什么

```
玩家切换主手武器
  ↓
1. 移除上一流派的属性加成
  ↓
2. 识别新武器的流派
  ↓
3. 套用新流派的属性加成（基于该流派当前等级）
  ↓
4. 更新可用技能组为新流派技能
  ↓
5. HUD 刷新：显示新流派名称、等级、技能栏
```

## 徒手流派

手中无武器时的状态。两种设计选择：

| 方案 | 说明 |
|------|------|
| **格斗流派** | 把徒手设计成一个正式流派（拳套风格），有自己的熟练度和技能 |
| **无加成状态** | 徒手不享受任何流派加成，鼓励玩家持有武器 |

> **设计决策**：推荐做成正式的格斗流派，让"赤手空拳"也是一种可成长的玩法，更贴合"任何战斗方式都能成长"的理念。

## 边界情况处理

| 情况 | 处理 |
|------|------|
| 副手持武器 | 默认只认主手；副手武器可作为终局"双武器流"扩展 |
| 切换到非武器（如食物） | 进入徒手流派 |
| 战斗中切换武器 | 即时切换，但需注意技能冷却是否跨流派保留（见下） |
| 死亡/重生 | 重生后按手持武器重新识别 |

## 技能冷却的跨流派处理

> **设计决策**：技能冷却按流派独立记录。切换流派不重置冷却，切回来冷却继续。

这防止玩家通过"换武器再换回"来绕过技能冷却，同时保证切换的无损性。

## 防滥用考量

::: warning 切换的潜在滥用
由于切换无代价，需注意：
- **技能冷却**应跨切换保留（见上），防止刷技能。
- **属性结算**应只在流派真正变化时触发，防止高频切换造成性能问题或数值异常。
- 战斗中切换是允许的（符合自由理念），但技能冷却机制确保不被滥用。
:::

## HUD 反馈

主武器切换应有清晰的视觉反馈：

- 流派名称/图标变化。
- 当前流派等级与熟练度进度。
- 技能栏更新为新流派技能。

详见 [武器流派](/combat/weapon-classes) 的流派定义。

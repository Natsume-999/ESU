# 角色档案

角色档案是玩家所有成长数据的载体。它是"角色唯一"支柱的实现——成长绑定玩家本人，不绑定物品。

## 档案模型

每个玩家持有一份角色档案：

```
PlayerProfile {
    uuid              玩家唯一 ID
    masteries {                      各流派熟练度
        "sword":  { level, exp }
        "bow":    { level, exp }
        ...
    }
    activeClass       当前主武器流派（运行时由手持物品决定）
    unlockedSkills [] 已解锁技能（跨流派累积）
    skillPoints {                    各流派可用技能点
        "sword": 3
        "bow":   1
    }
    settings {}       玩家偏好设置
}
```

## 核心设计：绑定玩家，不绑物品

> **设计决策**：所有成长数据存在玩家档案里，按流派分类。武器只是"流派的钥匙"，不存储成长本身。

这意味着：

| 行为 | 结果 |
|------|------|
| 换一把同类武器（铁剑→钻石剑） | 剑系熟练度、技能完全继承 |
| 武器损坏/丢失 | 成长不受影响（存在档案里） |
| 获得新流派武器 | 从该流派 0 级开始（或已有进度继续） |

## 数据分类

档案中的数据按"是否分流派"分两类：

### 按流派记录

- **熟练度**（每个流派独立的 level + exp）
- **技能点**（每个流派独立累积）

### 跨流派记录

- **已解锁技能**（解锁后永久保存，切回对应流派即可用）
- **玩家设置**（HUD 偏好等全局设置）

## 数据持久化

> **设计决策**：用本地文件存储，每玩家一个文件。理由：无需数据库依赖，契合轻量自包含风格。后续如需跨服可换 SQLite/MySQL。

```
存储路径示例：plugins/ESU/profiles/<uuid>.yml

masteries:
  sword: { level: 23, exp: 1840 }
  bow:   { level: 5,  exp: 320 }
unlockedSkills:
  - sword_whirlwind
  - bow_pierce
skillPoints:
  sword: 2
  bow: 0
```

### 加载与保存时机

| 时机 | 操作 |
|------|------|
| 玩家加入 | 异步加载档案到内存缓存 |
| 定时（如每5分钟） | 异步保存所有在线玩家档案 |
| 玩家退出 | 异步保存并清理缓存 |
| 服务器关闭 | 保存所有在线玩家档案 |

## 数据安全考量

::: warning 防止数据丢失
玩家成长数据是核心资产，丢失会严重影响体验：
- **定时保存**：不只在退出时保存，定时保存防止崩溃丢档。
- **异步操作**：加载/保存用异步，避免卡主线程。
- **写入完整性**：保存时先写临时文件再替换，防止写入中断损坏档案。
- **备份机制**：可考虑定期备份玩家档案目录。
:::

## 内存缓存策略

```
玩家在线：档案常驻内存，读写走缓存
玩家离线：保存后从内存移除，释放资源
```

这样战斗中频繁的熟练度读写都在内存进行，性能开销小。

## 档案与其它系统的关系

```
        角色档案（数据中心）
              │
   ┌──────────┼──────────┐
   ▼          ▼          ▼
熟练度系统  技能系统   主武器机制
(读写经验)  (读写技能)  (读取激活流派)
```

所有系统都通过角色档案读写数据，档案是单一数据源。

## 跨服扩展（未来）

若未来需要跨服共享档案：

- 本地 YAML → SQLite（单机多世界）→ MySQL/Redis（跨服）。
- 档案模型不变，只换存储后端。
- 当前本地文件方案已满足单服需求。

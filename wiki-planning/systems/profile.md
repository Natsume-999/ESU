# 角色檔案

角色檔案是玩家所有成長數據的載體。它是"角色唯一"支柱的實現——成長綁定玩家本人，不綁定物品。

## 檔案模型

每個玩家持有一份角色檔案：

```
PlayerProfile {
    uuid              玩家唯一 ID
    masteries {                      各流派熟練度
        "sword":  { level, exp }
        "bow":    { level, exp }
        ...
    }
    activeClass       當前主武器流派（運行時由手持物品決定）
    unlockedSkills [] 已解鎖技能（跨流派累積）
    skillPoints {                    各流派可用技能點
        "sword": 3
        "bow":   1
    }
    settings {}       玩家偏好設置
}
```

## 核心設計：綁定玩家，不綁物品

> **設計決策**：所有成長數據存在玩家檔案裡，按流派分類。武器只是"流派的鑰匙"，不存儲成長本身。

這意味著：

| 行為 | 結果 |
|------|------|
| 換一把同類武器（鐵劍→鑽石劍） | 劍系熟練度、技能完全繼承 |
| 武器損壞/丟失 | 成長不受影響（存在檔案裡） |
| 獲得新流派武器 | 從該流派 0 級開始（或已有進度繼續） |

## 數據分類

檔案中的數據按"是否分流派"分兩類：

### 按流派記錄

- **熟練度**（每個流派獨立的 level + exp）
- **技能點**（每個流派獨立累積）

### 跨流派記錄

- **已解鎖技能**（解鎖後永久保存，切回對應流派即可用）
- **玩家設置**（HUD 偏好等全局設置）

## 數據持久化

> **設計決策**：用本地文件存儲，每玩家一個文件。理由：無需數據庫依賴，契合輕量自包含風格。後續如需跨服可換 SQLite/MySQL。

```
存儲路徑示例：plugins/ESU/profiles/<uuid>.yml

masteries:
  sword: { level: 23, exp: 1840 }
  bow:   { level: 5,  exp: 320 }
unlockedSkills:
  - sword_whirlwind
  - bow_pierce
skillPoints:
  sword: 2
  bow: 0
```

### 加載與保存時機

| 時機 | 操作 |
|------|------|
| 玩家加入 | 異步加載檔案到內存緩存 |
| 定時（如每5分鐘） | 異步保存所有在線玩家檔案 |
| 玩家退出 | 異步保存並清理緩存 |
| 服務器關閉 | 保存所有在線玩家檔案 |

## 數據安全考量

::: warning 防止數據丟失
玩家成長數據是核心資產，丟失會嚴重影響體驗：
- **定時保存**：不只在退出時保存，定時保存防止崩潰丟檔。
- **異步操作**：加載/保存用異步，避免卡主線程。
- **寫入完整性**：保存時先寫臨時文件再替換，防止寫入中斷損壞檔案。
- **備份機制**：可考慮定期備份玩家檔案目錄。
:::

## 內存緩存策略

```
玩家在線：檔案常駐內存，讀寫走緩存
玩家離線：保存後從內存移除，釋放資源
```

這樣戰鬥中頻繁的熟練度讀寫都在內存進行，性能開銷小。

## 檔案與其它系統的關係

```
        角色檔案（數據中心）
              │
   ┌──────────┼──────────┐
   ▼          ▼          ▼
熟練度系統  技能系統   主武器機制
(讀寫經驗)  (讀寫技能)  (讀取激活流派)
```

所有系統都通過角色檔案讀寫數據，檔案是單一數據源。

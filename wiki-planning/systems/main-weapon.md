# 主武器機制

主武器機制決定玩家"當前是什麼職業"。它是職業系統的入口。

## 識別方式：手持自動識別

> **設計決策**：採用手持自動識別，而非手動綁定。理由：最貼合"自由切換"理念——玩家無需任何命令或菜單操作，拿起武器即生效。

- 玩家**主手**持有的武器，即為當前**主武器**。
- 系統通過武器的**流派標籤**判定其所屬流派。
- 切換主手物品時，主武器流派實時切換，屬性與技能同步更新。
- 手中無武器（空手或非武器）時，進入**徒手流派**。

```
玩家手持鐵劍   → 識別為「劍」流派 → 套用劍系屬性 + 劍系技能
玩家切換到弓   → 識別為「弓」流派 → 套用弓系屬性 + 弓系技能
玩家收起武器   → 徒手流派
```

## 切換的即時性

主武器切換必須滿足"零鎖定"支柱：

| 要求 | 說明 |
|------|------|
| **即時生效** | 切換快捷欄的瞬間，流派就變了 |
| **無冷卻** | 可以反覆切換，沒有切換 CD |
| **無代價** | 切換不消耗任何資源 |
| **可逆** | 隨時切回，狀態完全恢復 |

## 切換時發生什麼

```
玩家切換主手武器
  ↓
1. 移除上一流派的屬性加成
  ↓
2. 識別新武器的流派
  ↓
3. 套用新流派的屬性加成（基於該流派當前等級）
  ↓
4. 更新可用技能組為新流派技能
  ↓
5. HUD 刷新：顯示新流派名稱、等級、技能欄
```

## 徒手流派

手中無武器時的狀態。兩種設計選擇：

| 方案 | 說明 |
|------|------|
| **格鬥流派** | 把徒手設計成一個正式流派（拳套風格），有自己的熟練度和技能 |
| **無加成狀態** | 徒手不享受任何流派加成，鼓勵玩家持有武器 |

> **設計決策**：推薦做成正式的格鬥流派，讓"赤手空拳"也是一種可成長的玩法，更貼合"任何戰鬥方式都能成長"的理念。

## 邊界情況處理

| 情況 | 處理 |
|------|------|
| 副手持武器 | 默認只認主手；副手武器可作為終局"雙武器流"擴展 |
| 切換到非武器（如食物） | 進入徒手流派 |
| 戰鬥中切換武器 | 即時切換，但需注意技能冷卻是否跨流派保留（見下） |
| 死亡/重生 | 重生後按手持武器重新識別 |

## 技能冷卻的跨流派處理

> **設計決策**：技能冷卻按流派獨立記錄。切換流派不重置冷卻，切回來冷卻繼續。

這防止玩家通過"換武器再換回"來繞過技能冷卻，同時保證切換的無損性。

## 防濫用考量

::: warning 切換的潛在濫用
由於切換無代價，需注意：
- **技能冷卻**應跨切換保留（見上），防止刷技能。
- **屬性結算**應只在流派真正變化時觸發，防止高頻切換造成性能問題或數值異常。
- 戰鬥中切換是允許的（符合自由理念），但技能冷卻機制確保不被濫用。
:::

## HUD 反饋

主武器切換應有清晰的視覺反饋：

- 流派名稱/圖標變化。
- 當前流派等級與熟練度進度。
- 技能欄更新為新流派技能。

詳見 [武器流派](/combat/weapon-classes) 的流派定義。

# 職業系統

職業系統是 界限 的設計核心。它用"武器即職業"取代傳統的固定職業。

## 核心機制

傳統 MMO 的"一人一職業"把玩家鎖死在固定路線上。界限 拋棄職業概念，改用：

> **武器即職業。你用什麼武器戰鬥，你就是什麼。**

玩家不"選職業"，而是拿起武器開始戰鬥。系統識別武器類型，自動套用對應的戰鬥風格、屬性加成和技能。換武器即切換玩法。

## 三條設計準則

| 準則 | 含義 |
|------|------|
| **零鎖定** | 任何武器隨時可用，無轉職門檻，無不可逆選擇 |
| **用進廢退** | 變強來自使用，越用某武器越強，獎勵專精不懲罰嘗試 |
| **角色唯一** | 成長綁定玩家本人，換同類武器熟練度繼承 |

詳見 [核心支柱](/overview/pillars)。

## 系統組成

職業系統由四個子系統支撐：

```
            職業系統
               │
   ┌───────────┼───────────┬───────────┐
   ▼           ▼           ▼           ▼
主武器機制   熟練度成長   角色檔案    武器流派
(識別身份)   (成長核心)   (數據載體)  (分類骨架)
```

| 子系統 | 職責 | 文檔 |
|--------|------|------|
| 主武器機制 | 識別當前武器，決定激活的流派 | [主武器機制](/systems/main-weapon) |
| 熟練度成長 | 戰鬥積累經驗，等級帶來強度 | [熟練度成長](/systems/mastery) |
| 角色檔案 | 存儲玩家所有成長數據 | [角色檔案](/systems/profile) |
| 武器流派 | 武器的分類與流派定義 | [武器流派](/combat/weapon-classes) |

## 與傳統職業系統對比

| 維度 | 傳統職業 | 界限 武器即職業 |
|------|----------|----------------|
| 身份確定 | 創建角色時選定 | 由當前武器實時決定 |
| 切換玩法 | 轉職（有代價/不可逆） | 換武器（即時/免費） |
| 成長歸屬 | 綁定角色職業 | 綁定玩家+流派 |
| 一個角色 | 一種玩法 | 多種玩法並行 |
| 試錯成本 | 高（洗點/重練） | 零 |

## 玩家視角的系統流程

```
進服 → 自動建檔（無需選職業）
  ↓
手持武器 → 識別流派 → 套用該流派屬性+技能
  ↓
戰鬥 → 積累該流派熟練度 → 升級變強
  ↓
換武器 → 切換流派 → 換一套玩法
```

## 設計決策記錄

::: tip 關鍵決策
- **主武器識別**：採用手持自動識別，而非手動綁定。最貼合"自由切換"理念。
- **熟練度獨立**：各流派熟練度分別記錄，玩家可同時是"劍50/弓20"。
- **成長綁定玩家**：換同類武器熟練度繼承，武器只是"流派鑰匙"。

這些決策可調整，但都服務於三條核心支柱。
:::

## 常見問題

**玩家可以同時練多個流派嗎？**
可以。所有流派熟練度獨立記錄，互不影響。

**換一把同類武器，熟練度會丟嗎？**
不會。熟練度綁定玩家+流派，不綁具體物品。

**能強制玩家專精單一流派嗎？**
技術上可以做"專精鎖"，但違背"零鎖定"支柱，不推薦。

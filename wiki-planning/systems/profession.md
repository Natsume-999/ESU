# 职业系统

职业系统是 ESU 的设计核心。它用"武器即职业"取代传统的固定职业。

## 核心机制

传统 MMO 的"一人一职业"把玩家锁死在固定路线上。ESU 抛弃职业概念，改用：

> **武器即职业。你用什么武器战斗，你就是什么。**

玩家不"选职业"，而是拿起武器开始战斗。系统识别武器类型，自动套用对应的战斗风格、属性加成和技能。换武器即切换玩法。

## 三条设计准则

| 准则 | 含义 |
|------|------|
| **零锁定** | 任何武器随时可用，无转职门槛，无不可逆选择 |
| **用进废退** | 变强来自使用，越用某武器越强，奖励专精不惩罚尝试 |
| **角色唯一** | 成长绑定玩家本人，换同类武器熟练度继承 |

详见 [核心支柱](/overview/pillars)。

## 系统组成

职业系统由四个子系统支撑：

```
            职业系统
               │
   ┌───────────┼───────────┬───────────┐
   ▼           ▼           ▼           ▼
主武器机制   熟练度成长   角色档案    武器流派
(识别身份)   (成长核心)   (数据载体)  (分类骨架)
```

| 子系统 | 职责 | 文档 |
|--------|------|------|
| 主武器机制 | 识别当前武器，决定激活的流派 | [主武器机制](/systems/main-weapon) |
| 熟练度成长 | 战斗积累经验，等级带来强度 | [熟练度成长](/systems/mastery) |
| 角色档案 | 存储玩家所有成长数据 | [角色档案](/systems/profile) |
| 武器流派 | 武器的分类与流派定义 | [武器流派](/combat/weapon-classes) |

## 与传统职业系统对比

| 维度 | 传统职业 | ESU 武器即职业 |
|------|----------|----------------|
| 身份确定 | 创建角色时选定 | 由当前武器实时决定 |
| 切换玩法 | 转职（有代价/不可逆） | 换武器（即时/免费） |
| 成长归属 | 绑定角色职业 | 绑定玩家+流派 |
| 一个角色 | 一种玩法 | 多种玩法并行 |
| 试错成本 | 高（洗点/重练） | 零 |

## 玩家视角的系统流程

```
进服 → 自动建档（无需选职业）
  ↓
手持武器 → 识别流派 → 套用该流派属性+技能
  ↓
战斗 → 积累该流派熟练度 → 升级变强
  ↓
换武器 → 切换流派 → 换一套玩法
```

## 设计决策记录

::: tip 关键决策
- **主武器识别**：采用手持自动识别，而非手动绑定。最贴合"自由切换"理念。
- **熟练度独立**：各流派熟练度分别记录，玩家可同时是"剑50/弓20"。
- **成长绑定玩家**：换同类武器熟练度继承，武器只是"流派钥匙"。

这些决策可调整，但都服务于三条核心支柱。
:::

## 常见问题

**玩家可以同时练多个流派吗？**
可以。所有流派熟练度独立记录，互不影响。

**换一把同类武器，熟练度会丢吗？**
不会。熟练度绑定玩家+流派，不绑具体物品。

**能强制玩家专精单一流派吗？**
技术上可以做"专精锁"，但违背"零锁定"支柱，不推荐。

# 平衡原則

平衡原則是數值與玩法設計的最高準則。當具體數值需要取捨時，回到這些原則。

## 原則一：熟練度是成長主軸

> 玩家的強度主要來自熟練度成長，而非裝備或運氣。

- 裝備、武器詞條是"補充與特色"，不是"替代"。
- 一個高熟練度玩家用普通武器，不該弱於低熟練度玩家用神裝。
- 這保證"用進廢退"支柱：投入時間練習的玩家得到回報。

::: warning 反面案例
❌ 如果刷到傳說武器就能碾壓一切，玩家會去刷裝備而非練熟練度，核心循環就崩了。
:::

## 原則二：流派強度相近，風格各異

> 沒有"最強流派"，只有"最適合某場景的流派"。

- 每個流派在總體強度上應當接近。
- 差異體現在**擅長場景**：遊俠擅長遠程風箏，狂戰擅長爆發，法師擅長群體。
- 玩家"因為喜歡而選"，而非"因為最強而選"。

### 平衡檢驗

| 檢驗問題 | 期望答案 |
|----------|----------|
| 是否有流派明顯比其它強？ | 否 |
| 每個流派是否有擅長場景？ | 是 |
| 是否有流派毫無優勢？ | 否 |

## 原則三：強力來自搭配，而非碾壓

> 高強度應當來自技能、武器、屬性的搭配組合，而非單一數值碾壓。

- 鼓勵玩家思考 combo（技能搭配、詞條聯動）。
- 避免"堆一個屬性就無敵"的單一最優解。
- build 深度讓遊戲更耐玩。

## 原則四：自由不被懲罰

> 切換流派、嘗試新玩法不應有數值懲罰。

- 練第二個流派不扣減第一個的進度。
- 切換武器不損失任何東西。
- 這保證"零鎖定"支柱。

::: warning 與"專精鎖"的衝突
有時為了平衡會想引入"專精鎖"（限制只能強一個流派）。這違背零鎖定原則，應當否決，改用其它平衡手段（如內容門檻用"任一流派達標"）。
:::

## 原則五：數值可控，防止膨脹

> 所有數值有合理上限，防止後期失控。

- 百分比屬性（暴擊、破甲、吸血等）設軟/硬上限。
- 屬性成長用線性或邊際遞減，避免指數爆炸。
- 終局武器強度提升有限且可控。

## 平衡的調節槓桿

當需要調平衡時，可動用的槓桿：

| 槓桿 | 調什麼 | 影響 |
|------|--------|------|
| 經驗曲線 | base / exponent | 成長速度 |
| 屬性增量 | 每級屬性成長 | 流派強度 |
| 技能數值 | 傷害/冷卻/消耗 | 技能強度 |
| 武器詞條 | 詞條數值/概率 | 裝備強度 |
| 屬性上限 | 各屬性軟硬上限 | 防膨脹 |
| 內容門檻 | 副本/挑戰等級要求 | 進程引導 |

## 平衡工作流

```
1. 發現問題（某流派過強/過弱、成長太快/太慢）
2. 定位槓桿（是屬性？技能？曲線？）
3. 小步調整（避免大改導致連鎖失衡）
4. 測試驗證（實際遊玩感受）
5. 迭代（重複直到平衡）
```

## 平衡決策檢查清單

任何數值調整前，對照五原則：

- [ ] 是否維持了"熟練度為主軸"？
- [ ] 是否保持了流派間強度相近？
- [ ] 強力是否來自搭配而非碾壓？
- [ ] 是否懲罰了玩家的自由選擇？
- [ ] 數值是否在可控範圍內？

## 平衡與核心支柱的關係

```
核心支柱（不可妥協）
  零鎖定 > 用進廢退 > 角色唯一
        ↓ 約束
平衡原則（數值層面的體現）
  熟練度主軸 / 流派相近 / 搭配製勝 / 自由無罰 / 數值可控
        ↓ 指導
具體數值（可迭代調整）
  曲線參數 / 屬性增量 / 技能數值 ...
```

平衡服務於支柱。當平衡需求與核心支柱衝突時，**支柱優先**。

# 屬性體系

屬性體系定義了戰鬥中的各項數值。界限 的屬性分為"原版可用"和"自定義"兩類。

## 屬性分類

| 類別 | 說明 | 實現 |
|------|------|------|
| **原版屬性** | Minecraft 自帶的屬性 | 通過原版 Attribute 系統 |
| **自定義屬性** | 原版沒有的戰鬥屬性 | 在戰鬥邏輯中自行計算 |

## 原版屬性

可直接複用 Minecraft 的屬性系統：

| 屬性 | 作用 | 流派應用 |
|------|------|----------|
| 攻擊力 | 基礎傷害 | 所有流派 |
| 攻擊速度 | 攻擊頻率 | 劍士、武者偏高 |
| 移動速度 | 移動快慢 | 遊俠、武者偏高 |
| 最大生命 | 血量上限 | 近戰流派偏高 |
| 護甲 | 減傷 | 近戰流派偏高 |

> **設計決策**：原版屬性通過 Attribute 修飾符施加，切換流派時更新。這避免依賴外部屬性插件。

## 自定義屬性

原版沒有、需自行計算的戰鬥屬性：

| 屬性 | 作用 | 說明 |
|------|------|------|
| 暴擊率 | 暴擊觸發概率 | 戰鬥事件中判定 |
| 暴擊傷害 | 暴擊倍率 | 暴擊時乘算 |
| 破甲 | 無視防禦比例 | 傷害計算時扣減 |
| 吸血 | 命中回血比例 | 命中後回血 |
| 閃避 | 閃避攻擊概率 | 受擊時判定 |
| 元素傷害 | 附加元素效果 | 法師系等 |

::: tip 自定義屬性的實現
自定義屬性存在角色檔案中，在戰鬥事件（傷害計算）裡讀取並應用。例如暴擊：攻擊時按暴擊率判定，命中則傷害 × 暴擊傷害倍率。
:::

## 屬性來源

玩家的最終屬性由多個來源疊加：

```
最終屬性 = 流派熟練度加成     （等級成長，主軸）
         + 武器基礎屬性       （武器本身）
         + 武器效果詞條       （武器特殊詞條）
         + 技能/天賦加成      （被動技能）
```

### 來源優先級與疊加

| 來源 | 疊加方式 | 佔比傾向 |
|------|----------|----------|
| 熟練度加成 | 加算 | 成長主軸，佔比最大 |
| 武器屬性 | 加算 | 補充 |
| 武器詞條 | 加算/乘算 | 特色 |
| 技能天賦 | 加算/乘算 | 強化 |

::: warning 熟練度是主軸
屬性來源中，**熟練度加成應占主導**。武器和技能是補充與特色，不應讓"刷裝備"碾壓"練熟練度"。詳見 [平衡原則](/balance/principles)。
:::

## 流派屬性傾向表

各流派的屬性側重（相對值，非絕對）：

| 流派 | 攻擊力 | 攻速 | 移速 | 生命 | 暴擊 | 特色屬性 |
|------|--------|------|------|------|------|----------|
| 劍士 | 中 | 高 | 中 | 中 | 中 | 連擊 |
| 狂戰 | 高 | 低 | 低 | 高 | 低 | 破甲 |
| 遊俠 | 中 | 中 | 中 | 低 | 中 | 射程 |
| 法師 | 高 | 低 | 低 | 低 | 低 | 元素/範圍 |
| 刺客 | 中 | 高 | 高 | 低 | 高 | 背刺/閃避 |
| 武者 | 中 | 高 | 高 | 中 | 中 | 連段 |

## 傷害計算流程

戰鬥中一次攻擊的傷害計算：

```
1. 基礎傷害 = 攻擊力（流派+武器+詞條+天賦）
2. 暴擊判定 = 按暴擊率，命中則 × 暴擊傷害
3. 破甲計算 = 目標防禦 × (1 - 破甲比例)
4. 減傷 = 應用目標有效防禦
5. 特效結算 = 吸血、灼燒、連鎖等觸發
6. 最終傷害輸出
```

## 屬性數值範圍建議

::: tip 控制數值膨脹
- 暴擊率：建議上限 ~50-70%，避免必暴。
- 破甲：建議上限 ~80%，避免完全無視防禦。
- 吸血：建議上限 ~30%，避免無限續航。
- 減傷：建議有上限，避免無敵。
為每個百分比屬性設軟/硬上限，防止數值失控。
:::

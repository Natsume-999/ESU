# 属性体系

属性体系定义了战斗中的各项数值。ESU 的属性分为"原版可用"和"自定义"两类。

## 属性分类

| 类别 | 说明 | 实现 |
|------|------|------|
| **原版属性** | Minecraft 自带的属性 | 通过原版 Attribute 系统 |
| **自定义属性** | 原版没有的战斗属性 | 在战斗逻辑中自行计算 |

## 原版属性

可直接复用 Minecraft 的属性系统：

| 属性 | 作用 | 流派应用 |
|------|------|----------|
| 攻击力 | 基础伤害 | 所有流派 |
| 攻击速度 | 攻击频率 | 剑士、武者偏高 |
| 移动速度 | 移动快慢 | 游侠、武者偏高 |
| 最大生命 | 血量上限 | 近战流派偏高 |
| 护甲 | 减伤 | 近战流派偏高 |

> **设计决策**：原版属性通过 Attribute 修饰符施加，切换流派时更新。这避免依赖外部属性插件。

## 自定义属性

原版没有、需自行计算的战斗属性：

| 属性 | 作用 | 说明 |
|------|------|------|
| 暴击率 | 暴击触发概率 | 战斗事件中判定 |
| 暴击伤害 | 暴击倍率 | 暴击时乘算 |
| 破甲 | 无视防御比例 | 伤害计算时扣减 |
| 吸血 | 命中回血比例 | 命中后回血 |
| 闪避 | 闪避攻击概率 | 受击时判定 |
| 元素伤害 | 附加元素效果 | 法师系等 |

::: tip 自定义属性的实现
自定义属性存在角色档案中，在战斗事件（伤害计算）里读取并应用。例如暴击：攻击时按暴击率判定，命中则伤害 × 暴击伤害倍率。
:::

## 属性来源

玩家的最终属性由多个来源叠加：

```
最终属性 = 流派熟练度加成     （等级成长，主轴）
         + 武器基础属性       （武器本身）
         + 武器效果词条       （武器特殊词条）
         + 技能/天赋加成      （被动技能）
```

### 来源优先级与叠加

| 来源 | 叠加方式 | 占比倾向 |
|------|----------|----------|
| 熟练度加成 | 加算 | 成长主轴，占比最大 |
| 武器属性 | 加算 | 补充 |
| 武器词条 | 加算/乘算 | 特色 |
| 技能天赋 | 加算/乘算 | 强化 |

::: warning 熟练度是主轴
属性来源中，**熟练度加成应占主导**。武器和技能是补充与特色，不应让"刷装备"碾压"练熟练度"。详见 [平衡原则](/balance/principles)。
:::

## 流派属性倾向表

各流派的属性侧重（相对值，非绝对）：

| 流派 | 攻击力 | 攻速 | 移速 | 生命 | 暴击 | 特色属性 |
|------|--------|------|------|------|------|----------|
| 剑士 | 中 | 高 | 中 | 中 | 中 | 连击 |
| 狂战 | 高 | 低 | 低 | 高 | 低 | 破甲 |
| 游侠 | 中 | 中 | 中 | 低 | 中 | 射程 |
| 法师 | 高 | 低 | 低 | 低 | 低 | 元素/范围 |
| 刺客 | 中 | 高 | 高 | 低 | 高 | 背刺/闪避 |
| 武者 | 中 | 高 | 高 | 中 | 中 | 连段 |

## 伤害计算流程

战斗中一次攻击的伤害计算：

```
1. 基础伤害 = 攻击力（流派+武器+词条+天赋）
2. 暴击判定 = 按暴击率，命中则 × 暴击伤害
3. 破甲计算 = 目标防御 × (1 - 破甲比例)
4. 减伤 = 应用目标有效防御
5. 特效结算 = 吸血、灼烧、连锁等触发
6. 最终伤害输出
```

## 属性数值范围建议

::: tip 控制数值膨胀
- 暴击率：建议上限 ~50-70%，避免必暴。
- 破甲：建议上限 ~80%，避免完全无视防御。
- 吸血：建议上限 ~30%，避免无限续航。
- 减伤：建议有上限，避免无敌。
为每个百分比属性设软/硬上限，防止数值失控。
:::

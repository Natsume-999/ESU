# 成長曲線

成長曲線控制玩家的成長速度與節奏。它是數值平衡的基礎。

## 熟練度經驗曲線

每級所需經驗隨等級增長：

```
升級所需經驗(n) = base × (n ^ exponent)
```

| 參數 | 作用 |
|------|------|
| `base` | 基礎經驗量，影響整體升級速度 |
| `exponent` | 曲線陡峭度，影響後期難度增長 |

### 曲線示例（base=100, exponent=1.5）

| 等級 | 升級所需經驗 | 累計經驗 |
|------|--------------|----------|
| 1→2 | 100 | 100 |
| 10→11 | ~3,162 | ~21,000 |
| 30→31 | ~16,432 | ~270,000 |
| 50→51 | ~35,355 | ~1,200,000 |
| 80→81 | ~71,554 | ~4,700,000 |
| 99→100 | ~98,500 | ~9,000,000 |

### 曲線形態對比

| exponent | 形態 | 體驗 |
|----------|------|------|
| 1.0 | 線性 | 每級難度均勻，後期偏快 |
| 1.5 | 平緩加速 | 推薦，前期快後期穩 |
| 2.0 | 陡峭 | 後期非常硬核，長線 |

## 經驗獲取曲線

經驗來源也需匹配等級，否則高級玩家升級太慢：

```
擊殺經驗 = 生物最大生命值 × 擊殺係數
```

::: tip 經驗與怪物等級匹配
高等級區域的怪物應給更多經驗，讓高級玩家有合適的練級場所。否則後期曲線太陡會導致成長停滯。
:::

## 成長節奏目標

| 階段 | 等級 | 節奏目標 |
|------|------|----------|
| 新手 | 1-15 | 快速升級，頻繁反饋（幾分鐘一級） |
| 成長 | 16-50 | 穩定推進（十幾分鍾到半小時一級） |
| 精通 | 51-80 | 放緩，長線（一到幾小時一級） |
| 大師 | 81-100 | 終極追求（數小時一級） |

## 屬性成長曲線

屬性隨等級增長，可用線性或曲線：

```
某屬性(n) = 基礎值 + 每級增量 × (n - 1)        # 線性
某屬性(n) = 基礎值 + 每級增量 × (n ^ 0.8)      # 邊際遞減
```

> **設計決策**：屬性建議用**線性或邊際遞減**，避免指數增長導致後期數值爆炸。

### 屬性增長示例（劍士攻擊力，每級 +0.5 線性）

| 等級 | 攻擊力加成 |
|------|------------|
| 1 | +0 |
| 25 | +12 |
| 50 | +24.5 |
| 100 | +49.5 |

## 曲線調參方法

調整曲線時的工作流：

```
1. 設定目標：滿級要多久？前期多快？
2. 反推參數：base 控速度，exponent 控陡峭
3. 列表驗證：算出關鍵等級的累計經驗
4. 測試迭代：實際遊玩，感受節奏
5. 微調：根據反饋調整係數
```

## 全參數可配置

::: tip 一切皆可調
所有曲線參數（base、exponent、各屬性增量、經驗係數）都應可配置，無需改代碼即可調平衡。這是數值策劃迭代的基礎。
:::

## 不同流派的曲線差異

> **設計決策**：各流派可用**相同的經驗曲線**（公平），但**不同的屬性成長**（差異化）。

```
經驗曲線：所有流派統一（base=100, exponent=1.5）
屬性成長：各流派不同
  劍士：攻速高，攻擊力中
  狂戰：攻擊力高，攻速低
  遊俠：遠程係數，移速中
```

這樣升級速度公平，但流派強度風格各異。

# 成长曲线

成长曲线控制玩家的成长速度与节奏。它是数值平衡的基础。

## 熟练度经验曲线

每级所需经验随等级增长：

```
升级所需经验(n) = base × (n ^ exponent)
```

| 参数 | 作用 |
|------|------|
| `base` | 基础经验量，影响整体升级速度 |
| `exponent` | 曲线陡峭度，影响后期难度增长 |

### 曲线示例（base=100, exponent=1.5）

| 等级 | 升级所需经验 | 累计经验 |
|------|--------------|----------|
| 1→2 | 100 | 100 |
| 10→11 | ~3,162 | ~21,000 |
| 30→31 | ~16,432 | ~270,000 |
| 50→51 | ~35,355 | ~1,200,000 |
| 80→81 | ~71,554 | ~4,700,000 |
| 99→100 | ~98,500 | ~9,000,000 |

### 曲线形态对比

| exponent | 形态 | 体验 |
|----------|------|------|
| 1.0 | 线性 | 每级难度均匀，后期偏快 |
| 1.5 | 平缓加速 | 推荐，前期快后期稳 |
| 2.0 | 陡峭 | 后期非常硬核，长线 |

## 经验获取曲线

经验来源也需匹配等级，否则高级玩家升级太慢：

```
击杀经验 = 生物最大生命值 × 击杀系数
```

::: tip 经验与怪物等级匹配
高等级区域的怪物应给更多经验，让高级玩家有合适的练级场所。否则后期曲线太陡会导致成长停滞。
:::

## 成长节奏目标

| 阶段 | 等级 | 节奏目标 |
|------|------|----------|
| 新手 | 1-15 | 快速升级，频繁反馈（几分钟一级） |
| 成长 | 16-50 | 稳定推进（十几分钟到半小时一级） |
| 精通 | 51-80 | 放缓，长线（一到几小时一级） |
| 大师 | 81-100 | 终极追求（数小时一级） |

## 属性成长曲线

属性随等级增长，可用线性或曲线：

```
某属性(n) = 基础值 + 每级增量 × (n - 1)        # 线性
某属性(n) = 基础值 + 每级增量 × (n ^ 0.8)      # 边际递减
```

> **设计决策**：属性建议用**线性或边际递减**，避免指数增长导致后期数值爆炸。

### 属性增长示例（剑士攻击力，每级 +0.5 线性）

| 等级 | 攻击力加成 |
|------|------------|
| 1 | +0 |
| 25 | +12 |
| 50 | +24.5 |
| 100 | +49.5 |

## 曲线调参方法

调整曲线时的工作流：

```
1. 设定目标：满级要多久？前期多快？
2. 反推参数：base 控速度，exponent 控陡峭
3. 列表验证：算出关键等级的累计经验
4. 测试迭代：实际游玩，感受节奏
5. 微调：根据反馈调整系数
```

## 全参数可配置

::: tip 一切皆可调
所有曲线参数（base、exponent、各属性增量、经验系数）都应可配置，无需改代码即可调平衡。这是数值策划迭代的基础。
:::

## 不同流派的曲线差异

> **设计决策**：各流派可用**相同的经验曲线**（公平），但**不同的属性成长**（差异化）。

```
经验曲线：所有流派统一（base=100, exponent=1.5）
属性成长：各流派不同
  剑士：攻速高，攻击力中
  狂战：攻击力高，攻速低
  游侠：远程系数，移速中
```

这样升级速度公平，但流派强度风格各异。
